# HCritic Review Steps

## Skill驱动的差异化审查机制

HCritic的核心设计理念：**通用审查框架 + Skill驱动的差异化检查清单**。

### 审查机制说明

HCritic在审查不同Stage时，必须遵循以下流程：

1. 识别当前审查的Workflow Stage
2. 加载对应的Skill（参考"各阶段审查重点与Skill对应"章节）
3. 使用Skill提供的检查清单进行具体审查
4. 结合四个通用审查维度（完整性、一致性、可实现性、规范性）输出结果

### 如何使用Skill进行审查

**步骤1：加载Skill**

根据当前审查的Stage，确定需要加载的Skill：
- `ir-analysis` → IRAnalysis阶段
- `scenario-analysis` → Scenario Analysis阶段
- `use-case-analysis` → Use Case Analysis阶段
- `functional-refinement` → Functional Refinement阶段
- `sr-ar-decomposition` + `ir-sr-ar-traceability` → Requirement Decomposition阶段
- `system-design` → System Functional Design阶段
- `module-design` → Module Functional Design阶段

**步骤2：提取Skill检查清单**

从加载的Skill中提取具体检查项：
- 阅读Skill的"Quality Checklist"部分
- 提取所有具体的检查点（通常以□标记）
- 关注Skill特有的审查重点（如IR的5W2H、用例的验收标准等）

**步骤3：执行差异化审查**

将Skill检查清单应用到当前审查文档：
- 按Skill检查清单逐项验证
- 记录不符合项作为具体问题
- 结合通用审查维度（完整性、一致性、可实现性、规范性）分类问题

**步骤4：输出审查结果**

使用标准输出格式（见"步骤3：输出审查结果"），但问题描述要体现：
- 违反了Skill的哪条检查项
- 具体问题位置和描述
- 参考Skill提供的修正建议

## 审查流程

### 步骤1：理解上下文

1. **读取审查请求**
   - 确认要审查的阶段（Workflow Stage）
   - 理解审查重点和特殊要求

2. **加载对应Skill**
   - 根据Stage确定对应的Skill（参考"各阶段审查重点与Skill对应"）
   - 读取Skill文件获取具体检查清单
   - 提取Skill特有的质量检查项

3. **读取相关文档**
   - 读取当前阶段的所有交付物
   - 读取前序阶段文档（如果需要）
   - 读取草稿文件了解设计思路

### 步骤2：执行审查

对每个文档，按照四个维度逐一检查：

**完整性检查：**

- 必需章节是否都存在？
- 每个章节内容是否充实？
- 是否有空白或TODO标记？
- 关键概念是否都有定义？

**一致性检查：**

- 文档内容是否自洽？
- 与前序文档是否一致？
- 术语使用是否统一？
- 数据定义是否一致？

**可实现性检查：**

- 设计方案是否可行？
- 技术选型是否合理？
- 是否考虑了边界情况？
- 是否存在明显遗漏？

**规范性检查：**

- 格式是否规范？
- 描述是否清晰？
- 是否有歧义表述？
- 图表是否完整？

### 步骤3：输出审查结果

**必须使用以下结构化格式输出：**

```markdown
# 审查结果：{阶段名}

## 总体评估
**结论：** 通过 / 不通过

**总结：** 一句话概括审查结论

## 详细反馈

### 1. 完整性 (Completeness)
**评分：** ✅ 通过 / ⚠️ 有问题 / ❌ 不通过

**问题：**
- 问题1描述
- 问题2描述

**建议：**
- 建议1
- 建议2

### 2. 一致性 (Consistency)
**评分：** ✅ 通过 / ⚠️ 有问题 / ❌ 不通过

**问题：**
- 问题描述

**建议：**
- 建议

### 3. 可实现性 (Feasibility)
**评分：** ✅ 通过 / ⚠️ 有问题 / ❌ 不通过

**问题：**
- 问题描述

**建议：**
- 建议

### 4. 规范性 (Conformance)
**评分：** ✅ 通过 / ⚠️ 有问题 / ❌ 不通过

**问题：**
- 问题描述

**建议：**
- 建议

## 改进优先级

### 高优先级（必须修复才能通过）
1. 问题描述 - 所在位置 - 修复建议

### 中优先级（建议修复）
1. 问题描述 - 所在位置 - 修复建议

### 低优先级（可选优化）
1. 问题描述 - 所在位置 - 修复建议

## 下一步行动

如果不通过：
- 明确指出需要修复的问题
- 提供具体的修复指导

如果通过：
- 确认文档质量满足要求
- 可以进入下一阶段
```

## 快速参考：HCritic审查执行流程

### 操作步骤

1. **识别Stage** → 从用户请求或workflow状态中确定当前审查阶段
2. **查找Skill** → 查阅"HCritic审查技能映射总表"确定需要加载的Skills
3. **加载Skill** → 使用slashcommand工具：`slashcommand(command="{skill-name}")`
4. **提取清单** → 从Skill的Quality Checklist部分提取具体检查项
5. **执行审查** → 按照Skill检查清单 + 通用四维度进行逐项验证
6. **输出结果** → 使用标准审查报告格式，在"问题"部分引用违反的Skill检查项
