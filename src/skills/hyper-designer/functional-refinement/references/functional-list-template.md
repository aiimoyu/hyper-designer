# 功能列表文档模板

## 文档结构

**输出文件：** `{功能名}功能列表.md`

```markdown
# {系统名称} 功能列表

## 功能分析思路与结果总结

[简要描述分析方法：从用例到功能的映射，哪些用例需要新增功能，哪些用例修改现有功能]

示例：
- UC-001 用户登录 → 新增功能 F001（登录认证）
- UC-002 权限管理 → 修改功能 F005（用户管理），新增功能 F006（角色配置）


## 系统需求 (SR) 映射表

| 功能编号 | 功能名称 | 对应 SR 编号 | SR 描述 | 所属模块 (Bounded Context) | 优先级 |
|----------|----------|-------------|---------|----------------------------|--------|
| F001 | [功能名称] | SR-001 | [SR 描述] | M001 | Must |
| F002 | [功能名称] | SR-002 | [SR 描述] | M002 | Should |
**说明**:
- SR 编号格式: SR-NNN (如 SR-001, SR-002)
- 模块编号格式: M001, M002 (对应 DDD 限界上下文)
- 优先级: MoSCoW (Must/Should/Could/Won't)

## 非功能需求 (NFR/DFX) 汇总

| DFX 类别 | 来源 (上游输入) | 具体要求 | 量化指标 | 影响的功能编号 |
|----------|----------------|----------|----------|---------------|
| 性能 | IR 约束 / 场景需求 / 用例 DFX | [具体要求描述] | [量化指标] | F001, F002 |
| 安全 | IR 约束 / 场景需求 / 用例 DFX | [具体要求描述] | [量化指标] | F001, F003 |
| 可靠 | IR 约束 / 场景需求 / 用例 DFX | [具体要求描述] | [量化指标] | F002 |
| 隐私 | IR 约束 / 场景需求 / 用例 DFX | [具体要求描述] | [量化指标] | F001 |

**整合规则**:
1. **来源覆盖**: 从所有上游输入 (IR 约束、场景需求、用例 DFX 属性) 中提取 NFR/DFX 要求
2. **指标合并**: 当多个用例对同一 DFX 类别提出要求时，采用最严格的指标 (strictest-metric rule)
3. **功能影响**: 标注受影响的功能编号，确保设计时覆盖

**示例条目**:
| 性能 | UC-001 用例 DFX | 登录响应时间 | 响应时间<200ms (P95) | F001 |
| 安全 | IR 约束 | 密码传输加密 | TLS 1.3+ | F001, F003 |


如存在高风险功能，请参阅独立交付物：`{功能名}FMEA.md`
```

## 活动图分析模板（PlantUML）

**何时使用活动图：**
- 用例流程复杂，包含多个分支和判断
- 涉及多个系统或角色交互
- 需要清晰展示业务流程流转
- 异常处理路径较多

**PlantUML 活动图模板：**

```plantuml
@startuml {功能名称}_活动图
skinparam activity {
  BackgroundColor #F5F5F5
  BorderColor #333333
  FontSize 12
}

start
:步骤1 描述;

if (判断条件?) then (是)
  :分支A操作;
else (否)
  :分支B操作;
endif

:步骤2 描述;

switch (多分支判断)
case (条件1)
  :操作1;
case (条件2)
  :操作2;
case (其他)
  :默认操作;
endswitch

stop
@enduml
```

**带泳道的活动图模板（多角色协作）：**

```plantuml
@startuml {功能名称}_泳道活动图
|用户|
start
:用户操作;

|系统|
:系统处理;

if (验证条件?) then (通过)
  |数据库|
  :数据操作;
  |系统|
  :返回成功;
else (失败)
  :返回错误;
endif

|用户|
:查看结果;
stop
@enduml
```

## 示例：用户登录功能

```markdown
# 用户认证系统 功能列表

## 功能分析思路与结果总结

从用例 UC-001 用户登录 中提取功能：
- 主流程 → 新增功能 F001（登录认证）
- 异常流程 → 新增功能 F002（登录失败处理）
- 安全需求 → 新增功能 F003（会话管理）

### 登录流程活动图分析

```plantuml
@startuml 用户登录_活动图
skinparam activity {
  BackgroundColor #F5F5F5
  BorderColor #333333
}

|用户|
start
:输入用户名和密码;
:点击登录;

|系统|
:验证输入格式;

if (格式正确?) then (是)
  :查询用户信息;
  
  if (用户存在?) then (是)
    :验证密码;
    
    if (密码正确?) then (是)
      if (账号状态正常?) then (是)
        :创建会话;
        :记录登录日志;
        :返回登录成功;
        |用户|
        :进入主页;
      else (否)
        :返回账号异常;
      endif
    else (否)
      :记录失败次数;
      if (超过最大失败次数?) then (是)
        :锁定账号;
      endif
      :返回密码错误;
    endif
  else (否)
    :返回用户不存在;
  endif
else (否)
  :返回格式错误;
endif

stop
@enduml
```

## 功能描述

### F001 登录认证

**功能内容**: 验证用户身份，创建登录会话，返回认证结果。

**功能规则**: 
- 用户名：4-20位字母、数字或下划线
- 密码：8-20位，必须包含字母和数字
- 连续失败5次锁定账号30分钟

**功能约束**: 
- 会话有效期：24小时
- 同一账号同时只能在一个设备登录

**功能影响分析表格**:

| 功能编号 | 功能描述 | 影响类型 | 影响描述 |
|---------|---------|---------|---------|
| F001 | 登录认证 | 增 | 新增登录接口和会话管理 |
| F010 | 用户管理 | 改 | 需增加登录状态字段 |
| F011 | 日志记录 | 改 | 需增加登录日志类型 |

**优先级**: Must

### F002 登录失败处理

**功能内容**: 记录登录失败次数，超过阈值锁定账号。

**功能规则**: 
- 失败次数累计周期：30分钟
- 锁定时长：30分钟
- 锁定后需管理员解锁或等待自动解锁

**功能约束**: 
- 失败记录需持久化存储
- 支持管理员手动解锁

**功能影响分析表格**:

| 功能编号 | 功能描述 | 影响类型 | 影响描述 |
|---------|---------|---------|---------|
| F002 | 登录失败处理 | 增 | 新增失败记录表 |
| F010 | 用户管理 | 改 | 需增加锁定状态字段 |

**优先级**: Must

---

如存在高风险功能，请参阅独立交付物：`用户认证FMEA.md`
```
