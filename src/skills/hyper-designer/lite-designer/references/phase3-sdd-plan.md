# Phase 3 — SDD 开发计划

## 目标

将需求设计说明书中的 AR 分配拆解为**可直接执行的开发任务**，生成结构化的开发计划，使开发 Agent（如 Claude Code）能够无歧义地理解任务内容、执行顺序和验收条件。

---

## 前置条件

- 已完成 Phase 2，需求设计说明书已确认
- 所有 AR 的功能点已明确
- 修改围栏已确认

---

## Phase 3 开始前的交互节点

在开始拆分任务前，**一次性确认里程碑时间节点**：

```
在生成开发计划前，请确认：

1. 各里程碑的目标日期是否与 §4 中一致？
   M1（需求设计完成）：[日期]
   M2（P0 功能上线）：[日期]
   M3（P1 功能上线）：[日期]

2. 是否有并行开发的团队成员？如有，我可以标注哪些任务可以并行。

3. 有没有已知的技术风险需要在计划中提前预留时间？
```

### §6 给开发 Agent 的执行说明

> 以下说明直接面向 Claude Code 等开发 Agent：

**开始前必读**：
1. 读取需求设计说明书的"§约束：修改围栏"，确认可操作的代码范围
2. 按"§1.2 代码阅读清单"阅读现有代码，理解当前实现方式
3. 确认"§1.1 环境与依赖确认"中的所有前置条件已满足

**执行规范**：
- 按任务依赖顺序执行，不要跳过有前置依赖的任务
- 每完成一个任务，运行对应单元测试，全部通过后再继续下一个
- **遇到与设计不符的情况（如现有接口签名不一致、字段缺失等），先暂停并报告，不要自行修改设计方案**
- 新增代码的命名、注释风格遵循现有代码约定

**完成标志**：
- [ ] 所有任务的功能点清单全部勾选完成
- [ ] 所有单元测试通过
- [ ] 集成测试用例全部通过
- [ ] 回归测试无新增失败用例
- [ ] 代码 Review 通过（如有要求）
