## 模板使用说明

### 占位符规范

- `{xxx}`: 必填项，替换为实际内容
- `[xxx]`: 描述性说明，替换为实际内容
- `{NNN}`: 替换为三位数编号

### SR 数量说明

| SR 数量 | 建议 |
| :--- | :--- |
| 1 个 | 默认情况，所有功能归属同一系统能力 |
| 2 个 | 有明确的 Who / When / How Much 差异时拆分 |
| 3 个及以上 | 必须在分解前与用户确认每个 SR 的拆分理由 |

### 章节必填性

| 章节 | 必填性 | 说明 |
| :--- | :--- | :--- |
| 1. IR-SR 映射总览 | 必填 | 建立追溯链的起点 |
| 2. DDD 限界上下文划分 | 必填 | 模块边界定义 |
| 3. SR 分解 | 必填 | 核心交付内容 |
| 4. AR 分配 | 必填 | 核心交付内容 |
| 5. SR-AR 汇总表 | 必填 | 下游阶段直接引用 |
| 6. 接口与依赖矩阵 | 必填 | 系统元素间交互定义 |
| 7. 追溯链完整性自检 | 必填 | 质量门前置检查 |
| 8. 关键设计决策记录 | 推荐 | 设计理由的可追溯性 |
