# 访谈交互模式参考

## 核心原则

- **一次只问一件事**：避免信息过载
- **提供选项而非开放提问**：降低用户认知负担
- **每次交互后立即更新草稿**：防止信息丢失

## 开场模板

```
在【{阶段名}】中需要以下资料：{A}、{B}、{C}。
目前 {A} 有 {N} 份文件，{B} 有 {M} 份文件，{C} 暂缺。
我将按顺序逐项核对是否需要补充。如果某项没有相关文档，请直接告诉我"跳过"。
我们先从【{A}】开始……
```

## 场景应答

### 用户提供了资料

```
"收到。需要下载还是仅建立索引？是否有需要重点关注的文件或文件夹？"
```

处理后更新草稿，追加分析结果。

### 用户表示没有

```
"明白，已标记该项为[缺失]。这对后续任务可能产生一定影响，我们会稍后评估。
现在进入下一项：【{下一项}】……"
```

### 用户给出模糊回答

触发向下追问：

```
"具体是指哪个项目？有代码库链接或文档存档吗？我们主要参考它的架构还是业务逻辑？"
```

追问策略（按需选用）：

| 模糊类型 | 追问方向 |
| -------- | -------- |
| "参考以前的项目" | 哪个项目？代码库 or 文档？参考架构 or 业务逻辑？ |
| "网上有资料" | 具体 URL？关键词是什么？需要我搜索吗？ |
| "之前做过类似的" | 有产出物留存吗？在哪个目录/仓库？ |
| "找XX同事要" | 能否现在获取？还是标记为待补充？ |

### 用户提及互联网资源

使用子智能体获取：

- `task(subagent_type="librarian")`: 搜索 GitHub 仓库、技术文档
- `webfetch` / `websearch`: 获取公开资料

获取后在草稿中记录下载路径和内容摘要。

## 预扫描确认模板

```
预扫描完成，发现以下资料：

{资料类别A}:
  - file1.md — 描述1
  - file2.ts — 描述2

{资料类别B}:
  - file3.json — 描述3

请确认：
1. 以上分类是否正确？
2. 是否有需要重点关注的文件？
3. 是否有遗漏的文件需要补充？
```
