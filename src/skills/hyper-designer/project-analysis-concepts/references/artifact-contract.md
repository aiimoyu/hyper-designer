# Artifact Contract

This document defines the output contracts for all artifacts generated by the project-analysis workflow.

## Output Locations

### Human-Readable Artifacts

| Artifact | Location | Purpose |
|----------|----------|---------|
|
| `architecture.md` | `.hyper-designer/projectAnalysis/architecture.md` | System-level analysis with 5 architecture dimensions |
| `component/{componentSlug}.md` | `.hyper-designer/projectAnalysis/component/{componentSlug}.md` | Component-level analysis with 4 component dimensions |
| `coverage-report.md` | `.hyper-designer/projectAnalysis/coverage-report.md` | Coverage verification report with structured findings |

### Machine-Readable Manifests

| Manifest | Location | Stage | Purpose |
|----------|----------|-------|---------|
| `manifest.json` | `_meta/manifest.json` | Stage 1 | Canonical projectAnalysis metadata and rerun state |
| `system-analysis.json` | `_meta/system-analysis.json` | Stage 1 | System-level analysis summary |
| `component-manifest.json` | `_meta/component-manifest.json` | Stage 1 | Discovered component list and metadata |
| `api-manifest.json` | `_meta/api-manifest.json` | Stage 1 | Discovered API endpoints and interfaces |
| `source-inventory.json` | `_meta/source-inventory.json` | Stage 1 | Complete source code inventory |
| `components/{componentSlug}.json` | `_meta/components/{componentSlug}.json` | Stage 2 | Per-component analysis summary |
| `coverage-report.json` | `_meta/coverage-report.json` | Stage 3 | Structured coverage verdict and findings |

## Markdown Structure Requirements

### architecture.md

Must include:
- Project overview
- 5 architecture dimensions (see [architecture-dimensions.md](architecture-dimensions.md))
- Mermaid diagrams for:
  - Directory structure
  - Dependency graph
  - Data flow
  - State transitions
- Risk assessment
- Technology stack summary
- Update reminder section

### component/{componentSlug}.md

Must include:
- Component overview
- 4 component dimensions (see [component-dimensions.md](component-dimensions.md))
- Input/Output/Pos summary
- Mermaid diagrams for:
  - Internal dependency graph
  - Data flow within component
- Interface specifications
- Update reminder section

### coverage-report.md

Must include:
- Executive summary
- Coverage percentage
- 7 category check results
- Structured findings table
- Recommendations by severity
- Verdict (pass/fail)

## Contract Enforcement Rules

1. **All manifests must be valid JSON** - Stage 3 validates JSON structure
2. **All markdown files must follow structure** - Stage 3 validates section presence
3. **All Mermaid diagrams must be valid** - Stage 3 validates syntax
4. **All code citations must resolve** - Stage 3 validates file paths
5. **All cross-references must be valid** - Stage 3 validates links
6. **All update reminders must be present** - Stage 3 validates format

## Incremental Update Support

When re-running analysis:

1. Check if manifest exists
2. If exists, compare timestamps with source files
3. Only regenerate changed artifacts
4. Preserve unchanged manifests
5. Update `updatedAt` timestamp in manifest.json
