## 当前阶段：模块功能设计

**阶段标识**: `moduleFunctionalDesign`  
**执行Agent**: HEngineer  
**核心目标**: 为每个模块输出详细的技术规格，包括职责、接口、内部结构、算法、数据结构、测试策略。

### 1. 输入与资料收集

**在开始执行前，必须通过读取资料清单和自主搜集完成资料收集。**

所需的 **资料类别定义** 如下：

| 资料类别 | 关键内容 | 必需性 | 用途说明 |
| :--- | :--- | :--- | :--- |
| **系统设计资料**<br>(前阶段输出) | 系统设计文档（system-design.md） | 必需 | 系统架构、模块划分与接口契约 |
| **功能细化资料**<br>(功能列表) | SR 映射表、NFR/DFX 汇总表 | 必需 | §5 非功能要求实现的 NFR/DFX 参考依据 |
| **系统设计资料**<br>(模块参考) | 同类模块设计方案 | 可选 | 设计模式与最佳实践参考 |
| **代码库资料**<br>(接口规范) | API 设计规范、协议标准 | 可选 | 接口契约定义参考 |
| **代码库资料**<br>(开源实现) | 开源同类模块实现 | 可选 | 详细设计参考 |

**资料收集流程**（详见"单阶段处理流程 Step 2"）：

1. **读取资料清单**：读取项目根目录 `REFERENCE.md` 中相关 Section，解析用户填写的资料信息。
2. **确认完整性**：检查必需资料是否已填写，向用户汇报状态并确认是否需要补充。
3. **搜集与解析**：根据资料清单信息读取本地文件和URL，使用 `explore`/`librarian` 自主搜集补充资料，生成 `.hyper-designer/moduleFunctionalDesign/document/manifest.md`。

**核心参考资料**: `.hyper-designer/document/systemDesign/manifest.md` 和 `.hyper-designer/document/codebase/manifest.md`，由 HCollector 生成，包含已收集资料的结构化摘要和引用链接。

### 2. 执行规范与 Skill 使用

**核心 Skill**: `functional-design`

此 Skill 支持模块级详细设计，包括类设计、算法定义、接口规范与测试策略。

**模板遵循要求**：模块设计文档必须严格按照 `functional-design` Skill 中 `module-design.md` 定义的 §0–§9 章节结构输出：§0 概要、§1 目标与职责、§2 接口规范、§3 内部架构与组件、§4 数据结构与存储、§5 非功能要求实现、§6 测试与验证、§7 部署与运维注意事项、§8 开发注意与最佳实践、§9 附录。所有章节均不可省略。

### 3. 阶段交付物

| 文件名 | 路径 | 格式要求 |
| :--- | :--- | :--- |
| **{模块名}设计.md** | `.hyper-designer/moduleFunctionalDesign/{模块名}设计.md` | Markdown，包含完整 10 章节：§0 概要、§1 目标与职责、§2 接口规范、§3 内部架构与组件、§4 数据结构与存储、§5 非功能要求实现、§6 测试与验证、§7 部署与运维注意事项、§8 开发注意与最佳实践、§9 附录 |

**内容要点**：
- §0 概要：模块名称/ID、版本/作者/日期、依赖的系统功能与上游文档路径
- §1 目标与职责：模块目标、主要功能列表、边界与依赖
- §2 接口规范：对外 API、内部接口、事件发布/订阅
- §3 内部架构与组件：组件结构、类图、主要类/方法说明、关键算法与流程
- §4 数据结构与存储：核心实体定义、存储方案、索引策略、一致性保证
- §5 非功能要求实现：性能、安全、可用性（基于功能列表 NFR/DFX 汇总）
- §6 测试与验证：单元测试、集成测试、性能测试、验收标准
- §7 部署与运维注意事项：配置项、运行时依赖、日志/Tracing、常见故障自检
- §8 开发注意与最佳实践：可重用组件、代码风格、接口版本控制、依赖注入
- §9 附录：参考资料、设计模式应用、变更历史
