## 资料收集执行

读取 systemFunctionalDesign 阶段输出的系统设计文档，询问接口规范要求，搜索参考实现和设计模式，收集测试策略参考资料。

步骤示例：
1. 从工作流状态或阶段输出目录读取 system-design.md 或 system_design 文档（必须）。
2. 与需求方或接口拥有者确认接口契约（路径、方法、输入输出示例、错误码、版本兼容性）。
3. 在开源仓库和内部库中搜索参考实现与设计模式，收集有代表性的实现片段和架构决策记录。
4. 收集并整理测试策略参考（单元/集成/契约测试示例、性能基准、mock 数据模板）。

## 阶段输出

主输出：`{模块名}设计.md`（例如：auth-service设计.md）

内容要点：
- 模块概述与职责
- 公开接口（API/消息/事件契约）和示例
- 数据模型与重要类型定义
- 主要算法、流程与伪代码
- 错误与异常处理策略
- 部署、性能与运行时约束
- 测试策略与验收标准（单元、集成、契约、性能）

## 质量审核要求

审查要点：模块职责单一性、接口定义清晰性、测试策略完整性。

**强制审核流程：**

1. **生成文档**：完成本阶段所有输出文档
2. **立即触发HCritic审查**：文档生成后，**立即自动**使用 `HCritic` agent 进行设计审核
   - 调用方式：{{TOOL:delegate_critic_review}}
   - **不要等待用户确认，直接触发审查**
3. **处理审查结果**：
   - 审核未通过时：根据 HCritic 的反馈意见，**必须**修改文档并重新提交审查（返回步骤2）
   - 审核通过时：向用户确认"该阶段工作已完成，HCritic审核通过。是否进入下一阶段？"
4. **用户确认后**：执行 workflow 工具进入下一阶段
5. **禁止跳过审核**：未经 HCritic 审核通过的文档不得进入下一阶段
