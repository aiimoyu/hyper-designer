# 当前阶段

## 阶段信息

**阶段名称**: 场景分析 (Scenario Analysis)  
**执行Agent**: HArchitect  
**阶段目标**: 分析系统的使用场景，识别参与者、主场景、备选场景和异常场景，生成场景文档。

## 所需资料

在开始本阶段工作前，需要收集以下资料（基于 HCollector 4大类资料）：

| 资料类别 | 具体资料 | 用途 | 必需性 |
|---------|---------|------|-------|
| 领域资料 | 业务流程文档 | 现有业务流程图、操作手册 | 如有则必需 |
| 系统需求分析资料 | 场景库 | 用户场景、使用案例 | 如有则收集 |
| 系统需求分析资料 | 功能库 | 功能清单、需求规格 | 参考用 |
| 领域资料 | 用户角色定义 | 用户画像、角色权限 | 必需 |

### 资料收集执行

**执行步骤**：
1. 创建/更新 `.hyper-designer/document/draft.md`
2. 读取 IRAnalysis 阶段输出的 `需求信息.md`
3. 使用 Glob/LS 扫描项目中的相关资料
4. 与用户确认已发现的资料
5. 询问用户是否有业务流程文档或场景描述
6. 收集用户角色信息（谁会使用系统）
7. 生成资料索引 `.hyper-designer/document/manifest.md`

**提示**：使用 `document-collector` skill 指导资料收集流程

## 使用技能

**必须使用的 Skill**: `scenario-analysis`

此 skill 提供场景识别方法论、参与者分析指南，帮助系统化地识别和分析使用场景。

## 阶段输出

本阶段完成后应生成以下文档：

| 文档名称 | 路径 | 说明 |
|---------|------|------|
| {功能名}场景.md | `.hyper-designer/scenarioAnalysis/{功能名}场景.md` | 包含主场景、备选场景、异常场景的场景文档 |

## 质量审核要求

**审查要点**:

1. **完整性检查**: 主场景、备选场景、异常场景是否覆盖完整
2. **一致性检查**: Actor 定义是否清晰，场景间是否一致
3. **可追溯性检查**: 场景与需求信息是否对应

**强制审核流程：**

1. **生成文档**：完成本阶段所有输出文档
2. **立即触发HCritic审查**：文档生成后，**立即自动**使用 `HCritic` agent 进行设计审核
   - 调用方式：{{TOOL:delegate_critic_review}}
   - **不要等待用户确认，直接触发审查**
3. **处理审查结果**：
   - 审核未通过时：根据 HCritic 的反馈意见，**必须**修改文档并重新提交审查（返回步骤2）
   - 审核通过时：向用户确认"该阶段工作已完成，HCritic审核通过。是否进入下一阶段？"
4. **用户确认后**：执行 workflow 工具进入下一阶段
5. **禁止跳过审核**：未经 HCritic 审核通过的文档不得进入下一阶段
