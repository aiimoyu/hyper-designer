## 当前阶段：用例分析

**阶段标识**: `useCaseAnalysis`  
**执行Agent**: HArchitect  
**核心目标**: 将场景库中的功能场景细化为可执行、可验收的用例规格，明确输入/输出与验收标准。

### 1. 输入与资料收集

**在开始执行前，必须通过委派 HCollector subagent 完成资料收集。**

所需的 **资料类别定义** 如下：

| 资料类别 | 关键内容 | 必需性 | 用途说明 |
| :--- | :--- | :--- | :--- |
| **系统需求分析资料**<br>(前阶段输出) | 场景文档（{功能名}场景.md） | 必需 | 细化主场景、备选场景、异常场景 |
| **系统需求分析资料**<br>(风险参考) | FMEA库 | 可选 | 故障模式、异常处理场景参考 |
| **系统需求分析资料**<br>(功能参考) | 功能库 | 参考用 | 功能规格说明参考 |

**资料收集流程**：

1. **准备 HCollector 输入**：
   ```json
   {
     "stage": "useCaseAnalysis",
     "status": "init",
     "required_assets": [
       { "category": "系统需求分析资料(前阶段输出)", "description": "细化主场景、备选场景、异常场景（场景文档）" },
       { "category": "系统需求分析资料(风险参考)", "description": "故障模式、异常处理场景参考（FMEA库）" },
       { "category": "系统需求分析资料(功能参考)", "description": "功能规格说明参考（功能库）" }
     ]
   }
   ```

2. **委派 HCollector**：使用 `task` 工具调用 HCollector subagent 进行资料收集。

3. **多轮交互**：HCollector 将通过 JSON 响应请求与用户交互，你需要作为中继代理传递问答，直到 HCollector 返回 `action="finish"`。

4. **收集完成**：HCollector 将自动生成 `.hyper-designer/useCaseAnalysis/document/manifest.md` 和 `draft.md`。

详细的委派和交互协议请参见 **"单阶段处理流程 Step 2"**。

### 2. 执行规范与 Skill 使用

**核心 Skill**: `use-case-analysis`

此 Skill 将场景细化为详细的用例规格，定义用例的前置条件、成功保证、触发事件、Actor、主成功场景、扩展场景和DFX属性。

### 3. 阶段交付物

| 文件名 | 路径 | 格式要求 |
| :--- | :--- | :--- |
| **{功能名}用例.md** | `.hyper-designer/useCaseAnalysis/{功能名}用例.md` | Markdown，包含完整的用例规格（前置条件、触发事件、Actor、主成功场景、扩展/异常场景、验收标准） |

**交付节拍**：每个功能在完成后产出单文件

### 4. 质量审查

**审核标准:**

1. **用例规格完整性**: 前置条件、触发事件、Actor、主成功场景、扩展/异常场景是否完整且无歧义？
2. **输入/输出明确性**: 数据格式、类型、边界条件、错误代码/异常分支是否清晰定义？
3. **验收标准可测试性**: 验收条件是否能以自动化或手工测试判定通过/失败？
4. **规范性**: 文档结构是否符合 `use-case-analysis` skill 定义的模板？
