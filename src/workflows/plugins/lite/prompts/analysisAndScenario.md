## 当前阶段：需求分析与场景分析（轻量）

**阶段标识**: `analysisAndScenario`  
**执行Agent**: HArchitect  
**核心目标**: 在单模块范围内完成需求分析和场景分析，形成下游可直接使用的精简输入。

### 1. 执行规范与 Skill 使用

**核心 Skills**: `ir-analysis`, `scenario-analysis`

请结合 5W2H 与场景分类方法，完成最小充分分析。

### 2. 输出格式与长度限制（必须遵守）

输出文件：`需求场景分析.md`

```markdown
# 需求场景分析

## 1. 需求摘要
- 需求价值：[1-2 句话]
- 需求描述：[1-2 句话]
- 验收标准：[1-2 句话]

## 2. 5W2H
- What：[1-2 句话]
- Why：[1-2 句话]
- Who：[1-2 句话]
- When：[1-2 句话]
- Where：[1-2 句话]
- How：[1-2 句话]
- How Much：[1-2 句话]

## 3. 关键场景（最多 3 个）
### 场景 A
- 场景类型：业务/操作/维护/制造/其他
- 场景描述：[1-2 句话]
- 场景影响：[1-2 句话]
- 场景价值：[1-2 句话]

### 场景 B
...

### 场景 C
...

## 4. 下阶段输入摘要
- 单模块边界：[1 句话]
- 优先功能方向：[最多 3 条，每条 1 句话]
```

### 3. 质量要求

1. 不得输出超过 3 个关键场景。  
2. 不得把技术实现细节写入场景描述。  
3. 若信息缺失，提出最多 3 个微确认问题后再收敛输出。
