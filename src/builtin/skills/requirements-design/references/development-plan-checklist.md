# 开发计划检查表

本检查表用于评审开发计划的质量。

---

## §1 概要

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 需求来源是否标注 | ERROR | 需求分析说明书路径/版本 |
| 设计来源是否标注 | ERROR | 需求设计说明书路径/版本 |
| 项目类型是否正确 | WARNING | 全新项目 / 增量项目 |
| 任务总数是否明确 | WARNING | 开发任务数量 |
| 预估工作量是否合理 | INFO | 人日/人周估算 |
| 里程碑是否定义 | ERROR | M1/M2/M3 名称和目标日期 |
| 开发摘要是否清晰 | WARNING | 2-3 句描述整体任务范围 |

---

## §2 执行前准备

### 2.1 环境与依赖确认

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 开发分支创建说明 | WARNING | 分支命名规范 |
| 依赖列表是否完整 | WARNING | 依赖名称、版本、用途 |
| 数据库迁移权限 | INFO | 如有 schema 变更 |
| 外部服务访问权限 | INFO | 如有外部依赖 |

### 2.2 代码阅读清单

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 现有代码阅读清单 | WARNING | 列出需要阅读的文件及目的 |

### 2.3 修改围栏提醒

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 禁止修改区域是否明确 | 🔴 ERROR | 冻结路径及原因 |
| 允许修改区域是否明确 | ERROR | 允许路径及修改范围 |
| 条件修改是否说明 | WARNING | 需满足的条件/审批 |

---

## §3 开发任务列表

### 任务完整性检查

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| **每个 AR 是否有对应任务** | 🔴 ERROR | 所有 AR 必须有开发任务覆盖 |
| **任务编号是否规范** | WARNING | 格式为 T-{里程碑编号}{序号:02d} |
| **对应 AR 是否标注** | ERROR | 每个任务必须标注对应的 AR 编号 |
| **系统元素是否明确** | WARNING | 模块名及操作类型（新增/扩展） |
| **复杂度是否标注** | INFO | quick/medium/deep/integration |
| **依赖任务是否标注** | WARNING | 前置任务编号 |

### 任务内容检查

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 任务描述是否清晰 | ERROR | 2-3 句，从实现者视角说明 |
| 修改文件是否列出 | ERROR | 文件路径、位置、类型、内容 |
| 功能点清单是否完整 | ERROR | 直接来自 AR 功能点 |
| 实现要点是否说明 | WARNING | 关键步骤和注意事项 |
| 输出物是否明确 | WARNING | 新增/修改的文件 |

### 测试与验收检查

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 测试文件路径是否标注 | ERROR | 测试文件位置明确 |
| 正常路径测试场景 | ERROR | 至少 1 个正常路径 |
| 异常路径测试场景 | ERROR | 至少 1 个异常路径 |
| 边界条件测试 | INFO | 按需覆盖 |
| Mock 依赖是否说明 | WARNING | 需要 mock 的外部依赖 |
| QA 场景是否定义 | ERROR | 至少 2 个（正常+异常） |
| 验收命令是否可执行 | ERROR | 具体的测试命令 |
| 验收条件是否完整 | WARNING | 功能点、测试、覆盖率 |

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 测试文件命名规范 | WARNING | 命名规则说明 |
| 测试文件位置规范 | WARNING | 目录结构说明 |

| 覆盖率要求是否定义 | ERROR | 行覆盖率 > 80%，分支覆盖率 > 70% |

| 检查项 | 严重程度 | 检查要点 |

|--------|----------|----------|
| 集成测试是否定义 | WARNING | 覆盖主成功场景 |
| 验收命令是否明确 | WARNING | 可执行的测试命令 |

### 5.4 回归测试范围

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 回归测试范围是否定义 | WARNING | 现有模块的测试命令 |

## §6 验收标准

### 6.1 验收执行流程

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 修改围栏检测是否优先 | 🔴 ERROR | 围栏检测必须最先执行 |
| 验收流程是否完整 | ERROR | 单元测试→集成测试→QA场景→证据收集→覆盖率 |

### 6.2 修改围栏检测

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 检测方法是否说明 | ERROR | git diff 获取修改列表 |
| 围栏检测清单是否完整 | ERROR | 冻结模块、新增模块、修改范围、条件修改 |
| 围栏检测执行表 | WARNING | 冻结区域和允许修改区域的检测表 |
| 越界处理流程 | ERROR | 停止验收→记录→评估→处理→重新检测 |

### 6.3 验收通过标准

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 验收命令是否可执行 | ERROR | 具体的 npm 命令 |
| 预期结果是否明确 | WARNING | 每个命令的预期输出 |

### 6.4 验收失败处理

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 失败处理流程是否定义 | WARNING | 记录→分析→修复→重新验收 |

---

## §7 风险与注意事项

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|

| 编码规范是否说明 | WARNING | 命名规范、注释要求 |
| 关键实现注意 | INFO | 容易踩坑的地方 |

|--------|----------|----------|
