# 开发计划检查表

本检查表用于评审开发计划（SDD）的质量。检查项直接对应模板 `assets/development-plan-template.md` 的结构。

---

## §1 概要

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 需求来源是否标注 | ERROR | 需求分析说明书路径/版本 |
| 设计来源是否标注 | ERROR | 需求设计说明书路径/版本 |
| 项目类型是否正确 | WARNING | `Brand New Project / Incremental Project` |
| 任务总数是否明确 | WARNING | N 个开发任务 |
| 预估工作量是否合理 | INFO | N 人日 / N 人周估算 |
| 里程碑是否定义 | ERROR | M1/M2/M3 名称和目标日期 |
| 开发摘要是否清晰 | WARNING | 2-3 句描述整体任务范围、核心工作、交付目标 |

---

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 开发分支创建说明 | WARNING | `feature/[需求名]-[日期]` 格式规范 |
| 关键依赖与版本确认 | WARNING | 依赖名称、版本、用途 |
| 数据库/配置/外部服务权限 | INFO | 如有 schema 变更或外部依赖，需说明 |
| 其他前置条件 | INFO | 按实际项目调整 |

### 2.2 代码阅读清单

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 关键代码阅读清单是否列出 | WARNING | 修改前必须阅读的文件及目的（模块入口、数据模型、权限、错误处理等） |

### 2.3 修改围栏提醒

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 禁止修改区域是否明确 | ERROR | ⚠️ 路径及原因（来自设计文档） |
| 允许修改区域是否明确 | ERROR | ✅ 路径及修改范围 |

## §3 开发任务列表

### 里程碑结构检查

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 里程碑数量是否合理 | ERROR | ≤ 3 个里程碑 |
| 目标日期是否定义 | WARNING | 每个里程碑有目标日期 |

### 任务完整性检查

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| **每个 AR 是否有对应任务** | ERROR | 所有 AR 必须有开发任务覆盖 |
| **任务编号是否规范** | WARNING | 格式 `T-{里程碑编号}{序号:02d}`，如 `T-101` |
| **对应 AR 是否标注** | ERROR | 每个任务标注对应的 AR 编号/名称 |
| **系统元素是否明确** | WARNING | 模块名及操作类型（新增/扩展） |
| **复杂度是否标注** | INFO | `quick / medium / deep / integration` |
| **依赖任务是否标注** | WARNING | 前置任务编号（None 或 T-xxx） |

### 任务内容检查

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 任务描述是否清晰 | ERROR | 1-2 句，从实现者视角说明 |
| 修改文件是否列出 | ERROR | 文件路径、修改类型（新增/修改/删除）、修改说明 |
| 功能点清单是否完整 | ERROR | 直接来自 AR 功能点，可勾选 |
| 实现要点是否说明 | WARNING | 关键步骤，从实现者视角 |

### 测试与验收检查

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 测试与验收部分是否完整 | ERROR | 包含：单元测试、正常路径、异常路径、边界条件、Mock、测试命令 |
| QA 场景是否定义 | ERROR | 至少 2 个场景（正常路径 + 错误路径） |
| QA 场景格式是否规范 | WARNING | 包含：场景编号、类型、前置条件、步骤、验收命令、证据 |
| 完成标准是否完整 | WARNING | 功能点完成、测试通过、QA 场景通过、覆盖率达标 |

## 任务波次合理性检查

> 验证任务是否按框架优先、渐进细化原则分组

| 波次 | 内容 | 检查要点 |
|------|------|----------|
| Wave 0（全新项目） | 项目脚手架初始化、构建配置、目录结构、基础类型、核心依赖 | 全新项目必须有 |
| Wave 1 | 数据模型、接口类型、基础工具函数、配置文件 | 数据与接口层 |
| Wave 2 | 核心业务逻辑、服务层、业务规则处理 | 核心业务层 |
| Wave 3 | API 端点、UI 组件、模块间集成 | 接口与集成层 |
| Wave FINAL | 集成测试、E2E 测试、需求覆盖审计 | 验收层 |

**检查结论**：
- [ ] 全新项目包含 Wave 0 脚手架任务（必须）
- [ ] 任务按依赖和技术层级排序，不机械按 AR（警告）
- [ ] 耦合的 AR 合并到同一里程碑（建议）
