# 需求分析检查表

本检查表用于评审需求分析说明书的质量。

---

## §1 概要

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 名称是否明确 | ERROR | 名称应具体，能区分不同需求 |
| 描述是否清晰 | WARNING | 一句话说明核心目标 |
| 输入来源是否标注 | ERROR | 必须追溯来源（Issue/产品需求/口述） |
| 项目类型是否正确 | WARNING | 功能增强型 / 新功能开发 |

---

## §2 项目背景与 5W2H 分析

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 需求价值是否具体 | ERROR | 避免"提升用户体验"等泛泛描述，需追溯到业务根源 |
| 需求描述是否明确 | ERROR | 说明"要做什么"，范围清晰 |
| 5W2H 各项是否完整 | WARNING | 每项 1-2 句，聚焦核心信息 |
| Who 是否涵盖所有角色 | WARNING | 不要遗漏管理员角色 |

---

## §3 业务功能与场景

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 功能清单是否完整 | ERROR | 所有功能点均已列出，无遗漏 |
| 功能编号是否规范 | WARNING | 格式为 F-01, F-02... |
| 优先级是否合理 | ERROR | P0 功能必须实现，P1/P2 标注清晰 |
| 主成功场景是否描述 | ERROR | 用具名角色描述端到端操作流 |
| 业务场景分类是否完整 | WARNING | 业务/操作/维护/制造场景均已覆盖 |

---

## §4 性能规格

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 性能指标是否量化 | ERROR | 必须有具体数值（如 P95 < 500ms） |
| 测试条件是否明确 | WARNING | 并发数、数据量等条件已说明 |
| 是否有基线对比 | INFO | 如有改进需求，需说明基线 |

---

## §5 验收方法

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 验收标准是否可量化 | ERROR | 有数字、有基线、有测试方法 |
| P0 验收项是否完整 | ERROR | P0 验收项阻断发布 |
| 测试场景是否覆盖 | WARNING | 每个 P0 功能至少 2 个场景（正常+异常） |
| 交付物是否明确 | WARNING | 列出所有交付物及验收标准 |

---

## §6 约束

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 技术约束是否说明 | WARNING | 技术栈、接口协议、数据格式等 |
| 预算约束是否估算 | INFO | 开发资源、运维成本等 |
| 合规与安全要求 | WARNING | 权限、审计、数据安全等 |

---

## §7 边界

| 检查项 | 严重程度 | 检查要点 |
|--------|----------|----------|
| 做什么是否明确 | ERROR | In Scope 范围清晰，对应功能编号 |
| 不做什么是否明确 | ERROR | Out of Scope 显式标注，说明原因 |
| 范围蔓延风险 | WARNING | "看起来相关但本次不做"的事项已排除 |
