# 阶段 2: 组件分析

## 重要说明

本阶段是 **project-analysis** skill 的一部分。project-analysis 是一个项目分析工作流，包含3个阶段：
1. 系统分析
2. 组件分析（当前阶段）
3. 缺漏检查

在使用本阶段提示时，请确保理解整个 skill 的上下文和目标。

## 任务

深入分析阶段1中识别的每个模块/组件，生成单独的组件详情文件。

## 输入

- `overview.md` (阶段1)
- `architecture.md` (阶段1)
- `modules.md` (阶段1)

## 输出

`components/` 目录下的组件分析文件（如 `C001-Core.md`）

## 关键步骤

1. **评估分析深度**：根据依赖关系和模块职责分配优先级（高/中/低）
2. **确认分析范围**：若模块数量 > 8，与用户确认分析范围
3. **并行生成组件文档**：每个组件分配一个独立的分析任务
4. **验证文档**：确保所有 export 已列出，代码引用有行号，设计模式有证据

详细方法论见 `project-analysis` skill 的 `references/phase2-module-detail.md`
