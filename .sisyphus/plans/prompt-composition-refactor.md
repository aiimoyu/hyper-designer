# Prompt Composition Refactor: Split Prompts + Natural Language Tools

## TL;DR

> **Quick Summary**: Refactor agent prompt composition to use split prompt files (identity.md, constraints.md, step.md, workflow.md, standard.md, interview.md, tools) with natural language tool descriptions, remove {{TOOL:*}} placeholder system, and implement dynamic tools generation per CLI frontend.
>
> **Deliverables**:
> - Split prompt files for all agents (HArchitect, HCollector, HEngineer, HCritic)
> - Natural language tool usage templates replacing {{TOOL:*}} placeholders
> - Dynamic tools prompt generation via toolsGenerator per frontend
> - Remove placeholder resolver and toolRegistries
> - Update workflow_state.json with workflowId field
> - TDD implementation with comprehensive tests
>
> **Estimated Effort**: Large (multi-file refactor)
> **Parallel Execution**: YES - 3 waves
> **Critical Path**: Task 1 (Tests) → Task 2 (Remove placeholders) → Task 3 (Split prompts) → Task 4 (Tools integration) → Task 7 (Final verification)

### Original Request (Chinese)
用户希望重构提示词组合架构：

5. HArchitect已拆分，其他agent需仿照拆分

### Interview Summary

**Key Discussions**:
- Prompt composition: identity.md → constraints.md → step.md → workflow.md → standard.md → interview.md → tools (virtual)
- Workflow prompts: Hook-injected, not in promptFiles; selected from hd-config.json.workflow
- Tools prompts: Virtual, generated by function `generateToolsPrompt(frontend, toolList)`
- Two-layer approach: CLI-specific syntax (toolsGenerator) + natural language examples in agent prompts
- Replace {{TOOL:*}} with natural language templates in agent prompts
- HArchitect/HEngineer include workflow; HCollector/HCritic omit
- All agents get ask_user + task tools initially
- Frontend detected from plugin entry point (opencode plugin → opencode CLI)
- Test strategy: TDD (tests before implementation)
- Remove placeholder system entirely (resolver + toolRegistries)

**Research Findings**:
- Current: Agents use combined files (identity_constraints.md, interview_mode.md)
- HArchitect has unused split prompts in src/agents/HArchitect/prompts/
- toolsGenerator.ts exists but is not integrated
- Workflow prompts injected via OpenCode hook
- Placeholder system (resolver + toolRegistries) exists but will be removed

### Self-Review (Gap Analysis)

**Identified Gaps** (addressed in plan):
- Workflow prompt file naming: Use stage-specific files (e.g., IRAnalysis.md) in src/workflows/{workflow}/
- Agent promptFiles order: Must maintain consistent order across agents
- Tools virtual prompt injection point: Append after interview.md in factory
- workflowId persistence: Write on new state file initialization only
- Natural language template format: Use Chinese descriptions with English field labels
- Test coverage: Need tests for factory, toolsGenerator, and prompt content
